## Primary post (paste this)

**v0.7.4 is on main.** Biggest dashboard release yet: new sidebar IA, **Configuration** editors (stop hand-editing yaml for identity/radio/channels/MeshCore/MQTT/transmit/upstream), browser **Terminal**, **Settings → Updates** (check/apply/rollback from the UI), Auth tuning, MeshCore names+map fixes, experimental **native relay** (off by default). Builds on v0.7.3 login. No new pip packages.

**Navigation + polish.**
Sidebar: Dashboard / Messages / Stats / Radio (read-only) / Terminal / Configuration / Settings. Ctrl+K palette, themes, mobile fixes, noise-floor sparkline, unread-DM badge on Messages, 2h "online" window on nodes.

**Configuration.**
Edit almost everything from the dashboard. Radio tab is telemetry-only now. Node drawer gets battery/RSSI charts. °F/°C + miles/km toggles under Settings → Meshpoint.

**MeshCore + relay.**
Faster contacts, real names on map/Messages (timbot18), adverts carry position+signal. Optional SX1302 rebroadcast with original sender preserved (see docs before enabling relay).

Update your Meshpoint:

```
cd /opt/meshpoint
sudo git fetch origin
sudo git checkout main
sudo git pull origin main
sudo bash scripts/install.sh
sudo systemctl restart meshpoint
```

Hard-refresh the dashboard (Ctrl+Shift+R). First time on v0.7.3+? `/setup` for admin password. Old install (0.6.x / 0.7.2)? Use the full block above, not pull-only.

Or: admin login → Settings → Updates → Stable → Check → Apply.

**Thanks.** Community PRs in this release: **timbot18** (MeshCore friendly names + faster contact discovery), **iceice400** (MQTT hierarchical topic paths), **ottoxgam** (MeshCore channel config). **kendelmccarley**'s channel delete + frequency slot work from earlier merges is in the Configuration editors too. And everyone who ran the RC on real hardware.

Tested on RAK V2. Triangle update indicator within ~5 min.
