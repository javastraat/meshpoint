# v0.7.4 polish audit

This document is the agent's record of the four polish sprints
landed on top of the v0.7.4 feature work. It is intended to make
the design-pass visible without requiring a reviewer to read
every commit body.

## Scope

The v0.7.4 release adds the web terminal, dashboard-driven
update flow, MeshCore channel config, and a long list of auth
hardening tasks. The polish pass on top of that is internally
known as Sprints A through D.

## Sprint B — telemetry

Backend noise-floor pipeline, sidebar telemetry rail, reconnect
storyboard, live browser-tab title.

The noise-floor pipeline derives `noise_dBm = rssi - snr` from
incoming packets, smooths via EMA, clamps to a thermal-floor
plausibility window (`-174 + 10*log10(BW_Hz) + NF`), and
broadcasts a snapshot frame on the dashboard websocket every few
seconds. The sidebar sparkline subscribes to that frame.

Files added:

- `src/api/telemetry/noise_floor.py`
- `tests/test_noise_floor.py` (16 tests)
- `frontend/sidebar/telemetry_rail.js`
- `frontend/sidebar/noise_floor_sparkline.js`
- `frontend/sidebar/telemetry_rail.css`
- `frontend/js/reconnect_storyboard.js`

## Conventions kept

- Every new file has a top-of-file docstring stating its single
  responsibility.
- No file in this audit exceeds the 500-line cap; the heaviest
  is `frontend/js/command_palette.js` at ~190 lines.
- Each feature is a class behind a global instance; CSS lives in
  dedicated files alongside the JS.
- `prefers-reduced-motion` is honored where relevant
  (`route_motion.css`).
- `prefers-contrast: more` triggers the high-contrast bump when
  the user has not picked a theme.

## Test posture

The polish work is pure frontend except for the noise-floor
pipeline (Sprint B). Backend test count stayed at 614 passing
across all five commits. Frontend has no automated tests; manual
hardware validation on `.141` and `.15` is tracked separately in
`docs/plans/v0.7.4-tests/`.
