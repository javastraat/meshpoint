      Automatic checks

      Periodically checks for updates in the background using the same git-fetch check as "Check for updates" above, and shows a badge next to Settings in the sidebar when one's found.

      Authentication

      Manage admin credentials, sessions, the viewer role, and lockout policy.

      Change password

      Rotates this account's password and invalidates other browser sessions immediately.

      Sign out everywhere

      Invalidates every outstanding session, including this one. You'll be redirected to /login.

      Viewer role Disabled

      Read-only login for sharing the dashboard with trusted observers. Viewers cannot reach Settings or the Terminal.

      Failed-login lockout

      After this many failures, login is blocked for the cooldown window. Defaults to 5 attempts / 5 minutes.

      Session lifetime

      How long a fresh login stays valid before the dashboard bounces you back to /login. Pick a preset or set custom minutes (5 min – 30 days). Existing sessions keep their original expiry until they run out.

      Equivalent 8 hours

      System

      Display preferences for this browser and service-level actions that can leave the host in a degraded state.

      Display units

      Saved in this browser only. Node cards, the node drawer, and packet details use these units. Telemetry is stored in Celsius and meters on the device.

      Backup and restore

      Download a timestamped archive of config/local.yaml and the full data/ directory (SQLite snapshot, PKI keys, rollback state). The file contains all secrets. Store it offline. Do not commit it to git or email it.
